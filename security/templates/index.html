Texnoloji Analiz

01. Plain Text

Səviyyə: Sıfır Müdafiə

▼

Zəif Tərəfi:

Bazada sızma baş verdiyi an bütün parollar hər kəs tərəfindən oxuna bilər.

Nümunə:

İstifadəçi "admin123" parolu ilə daxil olur və bazada eynilə belə saxlanılır.

// INPUT: [user: "ali", pass: "123456"] ----> DATABASE: [user: "ali", pass: "123456"]

02. Hashing

Səviyyə: Orta Müdafiə

▼

Güclü Tərəfi:

Parollar birbaşa oxunmur, geri qaytarılması qeyri-mümkündür.

Zəif Tərəfi:

Rainbow Tables (hazır siyahılar) vasitəsilə sürətlə qırıla bilər.

// INPUT: [pass: "123456"] ----> DATABASE: [hash: "8d969eef6ecad3c29a3a629280e686cf0c3f5d5a86aff3ca12020c923adc6c92"]

03. Salting

Səviyyə: Yüksək Müdafiə

▼

Güclü Tərəfi:

Eyni parolu olan istifadəçilərin bazada hash kodları fərqli olur.

İşləmə Məntiqi:

Parolun sonuna "xG9!p" kimi unikal təsadüfi simvollar əlavə edilir.

// INPUT: [pass: "123456"] + [salt: "xG9!p"] ----> DATABASE: [hash: "e7cf4... (unikal nəticə)"]

04. Bcrypt

Səviyyə: Ən Müasir Seçim

▼

Güclü Tərəfi:

Brute-force (məcburi sınma) hücumlarını yavaşladan adaptiv "cost" faktoruna malikdir.

Nümunə:

Parolu yoxlamaq üçün 0.3 saniyə vaxt tələb edərək kütləvi hücumları qeyri-mümkün edir.

// INPUT: [pass: "123456"] ----> DATABASE: [hash: "$2y$12$K.rD5c.S... (duz daxilində, adaptiv)"]
